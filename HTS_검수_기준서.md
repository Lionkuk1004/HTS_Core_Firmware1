# HTS B-CDMA 검수 기준서 (전체 상세 항목)

본 문서는 HTS 3DPOWNS B-CDMA 펌웨어 전수검사 시 참조하는 상세 체크리스트이다.
프롬프트의 13항목 필수 검수 기준과 병행하여 아래 전체 항목을 적용한다.

---

## A. 동시성 오버헤드 및 메모리 배리어 (Concurrency & Memory Barriers)

### A-1. 오더링 오버헤드 최소화
- 기본값 std::memory_order_seq_cst는 전체 메모리 동기화 장벽 → 막대한 사이클 소모
- 상황에 맞춰 relaxed(독립적 카운터), acquire/release(생산자-소비자)로 다운그레이드했는지 점검

### A-2. 컴파일러/CPU 재배치 방어
- DMA/ISR 공유 메모리 영역에 std::atomic_thread_fence + volatile 조합
- 캐시 일관성(Cache Coherency) 유지, 파이프라인 최적화 순서 뒤바뀜 원천 차단

### A-3. Lock-free / Wait-free 설계
- ISR 내부/실시간 루프에서 mutex/spinlock 전면 배제
- 원자적 CAS(Compare-And-Swap) 기반 링 버퍼로 구현했는지 검증

---

## B. 메모리 및 리소스 무결성 (Memory & Resource Integrity)

### B-1. Zero-Copy 및 임시 객체 제거
- 함수 호출 시 const& 또는 포인터 사용, 불필요한 메모리 복사 제거
- 부팅 이후 런타임 동적 할당(new, malloc) 전면 금지

### B-2. 메모리 정렬(Alignment) 및 패딩(Padding)
- #pragma pack / alignas 남용·누락으로 버스 에러(Unaligned Access Fault) 발생 여부
- 캐시 라인 효율 저하 여부 → 구조체 배치 검사

### B-3. 스택(Stack) 무결성
- 대용량 로컬 배열 → static/전역 버퍼로 이동
- 깊은 재귀(Recursion) 배제 → 스택 오버플로우 차단

---

## C. 제어 흐름 및 실행 무결성 (Control Flow Integrity)

### C-1. CFI 검증
- 상태 머신 전이 시 이전 상태 유효성(Legal Transition) 매번 검증
- ROP/JOP 메모리 공격, 하드웨어 글리치 PC 변조 방어

### C-2. 예외 처리 배제
- -fno-exceptions에 맞춰 try-catch/throw 제거
- new(std::nothrow) 사용 → OOM 시 시스템 크래시 대신 에러 코드 반환

### C-3. 하드웨어 폴백(Fallback) 및 트랩
- HardFault/BusFault 발생 시 안전 에러 핸들러로 진입
- 레지스터 상태 로깅 + 리셋 규약 존재 확인

---

## D. 보안 및 안티포렌식 (Security & Anti-Forensics)

### D-1. Constant-time 실행
- 암호 키/서명/칩셋 판별 연산에 if/switch 조건 분기 배제
- 비트 마스크(&, |) 로직으로 타이밍 부채널 공격 차단

### D-2. 즉각적 보안 소거 (Secure Wipe)
- 민감 데이터(PRNG 키, 평문) 생명 주기 종료 즉시
- volatile 포인터 + memory_order_release로 0 소거, 컴파일러 DCE 차단

### D-3. 디버그 포트 무력화
- JTAG/SWD 접속 감지 → 워치독 만료 기다리지 않고 AIRCR 레지스터 즉시 타격
- 하드웨어 리셋 유발 확인

## E. DSP 및 알고리즘 오버헤드 제거 (Algorithm Optimization)

### E-1. ALU 병목 제거
- 나눗셈(/) / 모듈로(%) → 2의 거듭제곱 시프트(>>) + 비트 마스크(&) 100% 대체

### E-2. 하드웨어 Intrinsics 활용
- __builtin_popcount, __clz 등 단일 사이클 명령어 교체
- 소프트웨어 루프 비트 카운팅 제거

### E-3. 분기 예측 최적화
- Hot Path에 [[likely]] / [[unlikely]] 속성 부여
- 파이프라인 스톨 예방

## F. 데이터 무결성 및 통신 규약 (Data Integrity)

### F-1. 엔디안 독립성 보장
- 패킷/구조체 직렬화 시 포인터 캐스팅 금지
- 명시적 비트 시프트 연산으로 바이트 순서 오류 차단

### F-2. 오류 검출 코드

## G. 빌드 타임 검증 및 정적 분석 (Build-Time Verification)

### G-1. Static Assert

### G-2. 엄격한 형변환

### G-3. 경고의 에러화

## [본 기준서 적용 방법]

전수검사 시 각 파일에 대해:
1. 먼저 **프롬프트의 13항목 + ⑭PC코드 삭제**를 검사한다.
2. 이어서 **본 기준서의 A~S 상세 항목**을 해당 파일의 역할에 맞게 적용한다.
3. 위반 발견 시 즉시 수정하고, 검수 엑셀에 판정을 기록한다.
4. 임베디드 베어메탈 환경에 해당하지 않는 항목(SQL 인젝션, 파일 업로드 등)은 N/A 처리한다.
